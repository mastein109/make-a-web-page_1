Lesson 1: The Basics of Web and HTML

World Wide Web

The web includes many pieces that communicate with each other. A web page is a text document written in a language known as HTML. Web browsers read documents, interpret, and display them.

HTML

HTML stands for HyperText Markup Language and is the basis for everything on the web. HTML includes text content, markup, refereences to other documents, and links to other pages. For example: This is the link to the Udacity Homepage!

Vocabulary

Although it is not essential for programmers to memorize every little thing, there are a few terms that are necessary to know.
Tag: A tag is contained with brackets and usually consist of both an opening and closing tag. Tags that do not include content and do not include a closing tag are called "void tags." An example of a void tag is the image element.
Element: An element consists of everything within a set of opening and closing tags. An example of one element is the italics element.
Attribute: An attribute is a property of an HTML tag. For example, when I created the link to the Udacity homepage, I used the href attribute of an anchor tag.

Inline v Block

Certain elements are within text and are either applied to a specific line or create a new line. These are referred to as inline elements. Contrastly, elements that create and invisible box around content are considered block elements. For example, there are two different elements that create whitespace; break and paragraoph. Break tags insert a new line between the text, while paragraph creates a whole block of text that allows for attributes. For example, the italics element is considered an inline element.

HTML Documents

HTML documents provide a simple layout for a webpage. The layout contains tags, such as html, head, title and body. The head contains the title and styling like CSS and JS. Further, the body contains the main content of the entire webpage.

Lesson 2: Creating a Structured Document with HTML

Making A Webpage

HTML and CSS are both languages. However, HTML controls the structure of a webpage, while CSS controls the style. The structure of a webpage should be broken down into boxes or in HTML terms, div tags.
A Document Object Model or "DOM" referes to the tree-like structure of a page.

Common Errors in HTML and CSS

Check for syntax errors: Some of the most common and tedious errors made by programmers are typos and mistakes, such as closing an HTML element. Often times, it is easy to go back and fix the mistakes, but when it comes to large projects with extensive amounts of code, it is best to use a validation tool. The W3C CSS validator is best for these kinds of errors because the program will tell you exactly where your mistakes are located. In addition, you can use a linting tool, such as CSS Lint, which will help find errors.
Cross-browser Issues: Often times, your site looks fine in one browser, but there's problems in another. In order to avoid this you should always use a CSS reset, such as normalize. Normalize will minimize inconsistencies between browsers and make them uniform. Since HTML and CSS are always changing and improving, it is important to make sure HTML elements and CSS properties are supported. A good site to check on support for a certain element of property is caniuse.com.
Developer Tools: In the elements panel of the Chrome Developer Tools, you will find a computed tab. At the top of this tab, there is a box model diagram that is very important when it comes to how much space an element is occupying. The space is broken down into border, margin, padding, and content.

Lesson 3: Adding CSS Style

Cascading Style Sheets

CSS or the style part of HTML contains a key feature of inheritance. Inheritance relies on the ancestor-descendant relationship, which makes it possible for coders to avoid repetition. Further, CSS applies the most specific rule to every element, including its descendents. Descendents inherit text-related properties, but NOT box-related properties.

How Does CSS "control" HTML?

CSS utilizes a selector to apply rules to elements. Further, a class is then defined within a selector to a more specific class within an element.

Including Style

There are three different options when choosing where to include your style code.
1. Write CSS in the head section of HTML.
2. Link your HTML to separate CSS file. This is the most organized way to include your style in your HTML and avoid repetition*.
3. Write style inline with HTML. This is a bad idea because it most often leads to a lot of repeated code.

*Repetition: In software engineering, it is essential to not repeat yourself. If repetition is avoided in code, a modification of any single element does not cause a change in in multiple places of other unrelated elements.

The Box Model

The box model helps coders to breakdown the web page and style more efficiently. The different boxes include margin, border, padding, and content. The width of the border, padding, and content make up the element width.
